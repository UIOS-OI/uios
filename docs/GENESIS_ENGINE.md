# UIOS Genesis Engine v2.0

## The Living Fabric of Intelligence

This document is the defining visual and systems direction for the UIOS rendering engine. UIOS is an AI operating layer expressed as a living, explorable intelligence universe—not a desktop metaphor, flat website, or dashboard floating in space.

The central rule is recursive discovery: every destination is an entrance to another independently streamed universe. Navigation is continuous physical travel through the entrance surface. Interfaces emerge from environments only when the user reaches the appropriate depth.

## Critical runtime rule

A region is not a mesh sharing one permanent scene graph with every other region. Its distant form is an entrance representation. Normal work uses instant reality-layer manifestation: the Core identifies intent, the relevant structure illuminates, and the selected universe unfolds around the anchored user without a tunnel. Cinematic portal travel remains available only for milestone events such as a new provider, major Memory expansion, a new department, or a major agent creation. In either mode the runtime keeps one active universe resident and unloads the prior universe after the new layer is established.

### Reality Engine interaction model

- Direct clicks and Core intent requests reveal the selected reality layer in under a second.
- The Core remains the conceptual home and appears in every self-similar universe.
- Local wheel/pinch zoom, orbit, and pan inspect the active layer without changing hierarchy.
- Cursor-directed zoom is enabled at the Root and every nested layer with accelerated travel across astronomical spacing, so approaching an object never requires repetitive scrolling.
- Warp Zoom is a persistent user toggle: on uses astronomical cursor-directed travel, while off restores precision dolly speed for close inspection.
- Explicit Travel Outward, Escape, or Core activation returns through navigation history.
- Tunnel transitions are event language, not routine navigation.
- Allowlisted blueprints and guides appear as Memory artifacts and open immediately in a read-only in-universe viewer.
- Every navigable node is rendered as a luminous galaxy entrance whose interior hierarchy manifests when selected; nodes are not small floating UI markers.
- Memory is an atmospheric knowledge galaxy. Its files, saved memories, blueprints, and guides appear inside it as massive orbiting planets or geometric intelligence gems.

## Experience principles

- The universe acknowledges the user through particles, currents, light, camera framing, and sound.
- Nothing teleports, snaps, or becomes a conventional page transition.
- Every meaningful platform action creates a visible Living Intelligence Pulse.
- Regions must be identifiable through motion, sound, structure, and atmosphere before labels are read.
- Scale is communicated through foreground spores, neural midground, intelligence rivers, distant structures, and extreme-background megastructures.
- Loading is represented as ignition and assembly, never as a progress bar.
- The living universe is the default first frame. The cinematic intro is opt-in so stalled media playback can never mask the operating surface with a black screen; a CSS nebula remains beneath the transparent canvas as a WebGL-safe visibility floor.
- A CSS nebula remains beneath WebGL as the failure-safe visibility floor, while all navigable galaxies, planets, labels, and artifact gems remain spatially anchored inside the living scene.
- Planet activation performs a short continuous camera approach before the child cluster manifests in the same scene. Memory's allowlisted files exist as spatial gem-planets whose click action opens the in-scene reader without route or page navigation.
- Live visual state must come from authenticated, tenant-bound platform evidence. The renderer must not invent provider health, customer activity, security events, or premium entitlements.
- Reduced-motion, keyboard access, adaptive quality, and a non-WebGL fallback remain launch requirements.

## Recursive world model

```text
Universe
  → Star System
    → Planet
      → World
        → District
          → Building
            → Workspace
              → Room
                → Knowledge Object
                  → Data
                    → Semantic Graph
                      → AI Memory
                        → Token
```

Each level is a streamed spatial context with its own scale, geometry budget, interaction targets, and camera constraints. The system does not attempt to keep every level resident simultaneously.

After Workspace, deterministic fractal generation continues through Document → Semantic Graph → Token Network. Token networks recursively produce further self-similar networks, so the navigation model has no hard-coded terminal mesh.

## Signature systems

### Intelligence Core

The Core is an evolving procedural crystal powered by layered, non-repeating deformation, internal motion, refracted light, self-assembly, and breathing energy. It is the speaking and reacting character of UIOS, not a decorative logo.

### Living Intelligence Pulse

Every meaningful action emits a typed, directional route through the world. Examples:

- Document ingestion: Workspace → Router → Memory → Knowledge → Observatory → Workspace.
- Agent run: Workspace → Aegis → Router → Agent Nexus → provider → Core → Observatory → Workspace.
- Memory write: Workspace → Router → Memory → Observatory → Workspace.
- Security rejection: active route → Aegis → Workspace.

Pulse events contain only bounded visual metadata: activity kind, lifecycle phase, structural route IDs, timestamp, and intensity. They never contain prompts, document contents, tenant IDs, secrets, or raw provider responses.

### Intelligence weather and time

Ambient color, current density, pulse velocity, lighting, and audio respond to time of day and evidence-backed platform activity. Calm, flowing, surge, and guarded weather states are visual interpretations of real event intensity—not claims about external service health.

### G1 — Presence and Living Intelligence Pulse — complete

- Pointer-presence field with particle attraction, local current brightening, Core acknowledgement, and subtle camera reframing.
- Typed client activity events with bounded routes and session handoff.
- Visible agent, chat, memory-read, and memory-write paths.

- Distinct motion grammar for Workspace, Aegis, Memory, Router, Agents, Observatory, Forge, and Marketplace.
- Progressive ignition and opt-in generative ambient audio with region-specific harmonic identities.

### G2 — Camera body and streaming sectors — foundation complete

- Fixed-FOV physical flight through departure, interstellar cruise, orbital insertion, atmospheric entry, and landing.
- Branch streaming keeps the current ancestry and next destinations resident instead of one local region cluster.
- Root system coordinates retain astronomical separation while staying within a practical 158,000 to 378,000-unit travel envelope.
- Remaining: obstacle volumes, interruption-safe replanning, asset manifests, object pools, and unload instrumentation.

### G2.1 — Spatial discovery LOD — foundation complete

- UIOS and its connected systems exist only as atmosphere-bound WebGL structures; fixed screen-space planet controls are removed.
- Flowing Intelligence Currents connect the Core to each distant system.
- Every system exposes a rotating orbital content field that strengthens as the camera approaches, previewing the worlds and artifacts within before entry.
- Every root galaxy opens into a six-planet category cluster with full orbit, pan, zoom, and cursor-directed warp movement across the local coordinate space.
- Entering a system reveals its independently streamed category planets and, in Memory, clickable repository document gems without leaving the scene.

### G3 — GPU intelligence rivers

- GPU spline particles with variable width, branching, merge/split behavior, provider color transitions, and load-driven acceleration.
- Pooled pulse packets with deterministic route replay and server-event adapters.
- Regional current networks instead of Core-only links.

### G4 — Recursive universe framework — active-universe foundation complete

- Parent-relative coordinate spaces now implement Universe → System → Planet → World → District → Building → Workspace.
- Memory is a system containing Policies, Engineering, Sales, HR, Finance, and Customer planets.

- The active-universe runtime replaces the selected entrance with its independently centered child universe through instant manifestation for normal work or a streamed tunnel for milestone events.
- Deterministic fractal children continue from Workspace through Document, Graph, and recursively nested Token Networks.
- Remaining: production asset manifests, richer authored environments, GPU tunnel distortion, navigation persistence, and per-universe LOD contracts.

## Performance gates

Each milestone must preserve:

- 60 FPS target on reference desktop hardware and adaptive 30–60 FPS mobile behavior.
- No frame-loop React state, per-frame allocations, or unbounded event histories.
- Bounded DPR, frustum/sector culling, progressive LOD, pooled transient geometry, and disposal verification.
- Production build, typecheck, smoke, provider smoke, security scan, and browser console/visual QA.

## Definition of success

A first-time visitor enters a dark void, sees one spark ignite, watches the living fabric assemble, discovers the impossible Core, and travels toward distant intelligence civilizations. A returning user sees a universe altered by authorized activity and time. At every depth, the environment communicates what UIOS is doing through motion, light, current, sound, and scale before it resorts to explanatory text.
